# 运行说明

## 环境要求

- JDK 11 或以上
- Maven 3.6 或以上
- Redis 5/6，端口默认 `6379`
- RabbitMQ 3，端口默认 `5672`
- 当前 Swing `View` 项目保持不动

## 服务配置

默认配置位于 `InspectionBackend/common/src/main/resources/application.properties`。

```properties
redis.host=localhost
redis.port=6379
redis.database=9
redis.password=

rabbitmq.host=localhost
rabbitmq.port=5672
rabbitmq.username=Mio123
rabbitmq.password=Mio123
rabbitmq.virtualHost=Mio123
```

也可以用环境变量覆盖，例如 `REDIS_HOST`、`RABBITMQ_HOST`、`RABBITMQ_USERNAME`、`RABBITMQ_PASSWORD`、`RABBITMQ_VHOST`。

## 初始化

在 `InspectionBackend` 根目录执行：

```bash
mvn clean package -DskipTests
mvn -pl tools -am exec:java -Dexec.mainClass=edu.homework.inspection.tools.DefaultDataTool -Dexec.args=init
```

该命令会：

- 声明 RabbitMQ 交换机和队列。
- 写入默认登录用户。
- 保持 Redis DB 9 与现有 `View` 的读取方式一致。

如需清理运行数据：

```bash
mvn -pl tools -am exec:java -Dexec.mainClass=edu.homework.inspection.tools.DefaultDataTool -Dexec.args=clean-runtime
```

## 一键启动

```bash
mvn -pl launcher -am exec:java -Dexec.mainClass=edu.homework.inspection.launcher.LauncherMain
```

启动后端后，再启动 `View/CREAZYTHURSDAY/com.Manny` 中的 `Login.Main`。

推荐验收流程：

1. 用 `config/config123` 登录。
2. 选择默认地图或自定义地图。
3. 添加小车和障碍物。
4. 选择 A*、Bidirectional A* 或 Dijkstra。
5. 点击“开始探索”。
6. 地图中应能看到小车逐步移动并点亮区域。
7. 用 `analyst/analyst123` 登录回放界面，点击回放并验证速度和进度控制。

## 独立进程启动

每个知识源都可单独启动：

```bash
mvn -pl controller -am exec:java -Dexec.mainClass=edu.homework.inspection.controller.ControllerMain
mvn -pl navigator -am exec:java -Dexec.mainClass=edu.homework.inspection.navigator.NavigatorMain
mvn -pl car -am exec:java -Dexec.mainClass=edu.homework.inspection.car.CarMain
mvn -pl recorder -am exec:java -Dexec.mainClass=edu.homework.inspection.recorder.RecorderMain
```

这种方式用于展示“分布式独立进程部署”。

## 常见问题

- 登录失败：先运行 `tools init`，确认 Redis DB 9 中存在 `Users:config` 等 key。
- 点击开始无反应：确认 RabbitMQ vhost/user/password 与配置一致。
- 小车不移动：确认已在 View 中添加小车，Redis 中存在 `Cars:1` 等 key。
- 回放无记录：需要先完成一次探索并由 `recorder` 记录到 `Record:*`。
