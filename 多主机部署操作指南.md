# 多主机部署操作指南

## 架构说明

只有一台主机运行 Redis + RabbitMQ + 后端，其余主机只需运行 View 前端。

```
主机 A（服务端）                    主机 B/C/D（客户端）
┌──────────────────┐              ┌──────────────┐
│  Redis :6379     │◄─────────────│              │
│  RabbitMQ :5672  │   局域网连接   │  View 前端    │
│  后端全部模块     │              │  (只跑前端)   │
└──────────────────┘              └──────────────┘
```

---

## 一、主机 A 准备工作（服务端）

### 1. 查看本机 IP

打开 CMD，输入：

```bash
ipconfig
```

找到 `IPv4 地址`，例如 `192.168.233.1`。把这个 IP 告诉其他同学。

### 2. 创建 RabbitMQ 远程用户

> ⚠️ `guest` 账号只能本机登录，必须创建新用户！

```bash
# 创建用户
rabbitmqctl add_user 用户名 密码

# 授予管理员权限
rabbitmqctl set_user_tags 用户名 administrator

# 授予虚拟主机访问权限
rabbitmqctl set_permissions -p / 用户名 ".*" ".*" ".*"
```

示例：
```bash
rabbitmqctl add_user team3 team3123
rabbitmqctl set_user_tags team3 administrator
rabbitmqctl set_permissions -p / team3 ".*" ".*" ".*"
```

### 3. 确认 Redis 无密码（默认）

```bash
redis-cli ping
```

返回 `PONG` 即正常。如要求输入密码，查看 `redis.conf` 中的 `requirepass`。

### 4. 关闭防火墙（或放行端口）

**方式一：临时关闭防火墙（最简单）**
```
控制面板 → Windows Defender 防火墙 → 启用或关闭 → 关闭
```

**方式二：放行指定端口**
```bash
# 以管理员身份运行 CMD
netsh advfirewall firewall add rule name="Redis" dir=in protocol=tcp localport=6379 action=allow
netsh advfirewall firewall add rule name="RabbitMQ" dir=in protocol=tcp localport=5672 action=allow
```

### 5. 验证端口已开放

```bash
netstat -ano | findstr "6379"
netstat -ano | findstr "5672"
```

看到 `0.0.0.0:6379` 和 `0.0.0.0:5672` 的 `LISTENING` 状态即为成功。

## 二、主机 B/C/D 操作（客户端，只跑前端）

### 1. 确保在同一局域网

- 连接同一个 WiFi
- 或者用手机热点，大家都连同一个热点

### 2. 测试连通性

先测试能否 ping 通主机 A：

```bash
ping 192.168.233.1
```

再测试 Redis 和 RabbitMQ 端口：

```bash
redis-cli -h 192.168.233.1 ping
```

返回 `PONG` → 连通。

### 3. 启动 View 前端

双击 `start-view.bat`，按提示输入：

| 提示 | 填什么 | 示例 |
|------|--------|------|
| `Server IP` | 主机 A 的 IP | `192.168.233.1` |
| `Redis password` | 直接回车 | (无密码) |
| `RabbitMQ user` | 主机 A 创建的账号 | `team3` |
| `RabbitMQ password` | 主机 A 设置的密码 | `team3123` |

## 三、常见问题排查

| 现象 | 原因 | 解决 |
|------|------|------|
| 登录界面弹不出 | Redis 连不上 | 检查主机 A 防火墙 |
| 登录成功但点"开始巡检"无反应 | RabbitMQ 连不上 | 检查 RabbitMQ 用户名密码 |
| `guest can only connect via localhost` | 用了 guest 账号 | 换成主机 A 新创建的账号 |
| ping 不通 | 不在同一网络 | 检查是否连了同一 WiFi |
| `redis-cli` 无此命令 | 没装 Redis 客户端 | 把主机 A 的 `redis-cli.exe` 复制过来即可 |
